# Request Execution Flow

يوضح هذا المستند كيف تتدفق البيانات والطلبات من واجهة المستخدم وحتى إتمام تنفيذها:

[UI Trigger] 
      ↓
[Mediator.send(Command/Query)]
      ↓
┌────────────────── Validation Pipeline ──────────────────┐
│  1. PermissionValidator -> (يفحص الصلاحيات الإدارية)      │
│  2. PlatformValidator   -> (يفحص تحديد المنصات)         │
│  3. Title/MediaValidator-> (يفحص هيكلية البيانات المتدفقة)│
└─────────────────────────────────────────────────────────┘
      ↓
┌───────────────────── Policy Engine ─────────────────────┐
│  PublishPolicy.canPublish()                             │
│  -> (يفحص توافق الكيانات مع شروط وقوانين منصات التواصل)  │
└─────────────────────────────────────────────────────────┘
      ↓
[Transaction Open] (عند الحاجة لحماية العمليات المتسلسلة)
      ↓
[RequestHandler.handle()] -> (تنفيذ منطق العمل الفعلي)
      ↓
[Mappers] -> (تحويل الـ Entity إلى DTO المخصص للمنصة الخارجية)
      ↓
[Infrastructure / API Client] -> (إرسال البيانات للسيرفرات)
